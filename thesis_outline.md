# Rust 非正常控制流下的内存安全问题研究
Research on Memory Safety Issues in Rust Under Abnormal Control Flow

## 摘要与关键词
- 中文摘要（与 `abstract.md` 一致）
  - 背景：Safe Rust 默认内存安全；panic/unwind 下异常控制流与 Drop、unsafe、FFI 交织引入双重释放、UAF、跨边界 UB 等风险
  - 问题：panic/展开语义下分类标准不统一、评测不系统、工具能力边界不清
  - 方法：（1）触发点—资源状态—隔离边界三维度场景分类（S1–S4）及机理分析；（2）基于场景分类的静态分析评测框架，在统一用例集上评测 Rudra、MirChecker、FFIChecker
  - 结果：基础指标 + 分场景 Recall 等量化对比；S3 上 FFIChecker 相对通用工具的反差、工具间互补与短板（摘要中给出具体数值）
  - 贡献：第 3 章 基于状态碎化理论的异常控制流多维场景分类；第 4 章 评测框架、实证结论与可复现流程
- 关键词：Rust；panic；非正常控制流；Drop；unsafe；FFI；静态分析；能力边界；漏洞挖掘

## 3 基于状态碎化理论的异常控制流多维场景分类体系（创新点一，`chapter3_classification.md`）
### 3.1 引言
- 问题界定：第三章仅聚焦“100 个漏洞样例的场景分类”而非工具测评
- 研究目标：验证分类体系的可判别性、可解释性与边界稳定性
- 核心思路：基于代码/接口层证据构建决策树分类，并通过节点消融分析贡献

### 3.2 方法设计
- 3.2.1 状态碎化理论与分类目标：基本异常保证、状态碎化、三阶段演进（触发/碎化/物化）
- 3.2.2 决策树分类器：ABI 跨域 → S3；并发独占 → S4；单线程扩张期/消亡期 → S2/S1
- 3.2.3 代码/接口层证据定义：FFI 边界证据、析构阶段证据、资源初始化阶段证据、并发同步证据
- 3.2.4 判定流程与输出：证据抽取 → 节点判定 → 场景标签（S1–S4）→ 边界样例记录
- 3.2.5 节点贡献消融：`step1_only`、`step1_step2`、`step1_step3`、`full` 等变体

### 3.3 实验环境与设置
- 3.3.1 数据集：`testcases/testcase_categories.csv` 的 100 例（S1=4, S2=66, S3=16, S4=14）
- 3.3.2 证据抽取对象与规则：以 `testcases/` 源码与接口形式为主，不依赖人工随机设定
- 3.3.3 指标体系：场景召回、宏平均 F1、混淆矩阵、边界错误对
- 3.3.4 消融设置：节点缺失与证据可见性变化下的分类性能比较

### 3.4 实验结果与分析
- 3.4.1 完整决策树结果：总体宏平均 F1 与分场景召回
- 3.4.2 节点消融结果：比较各变体性能下降幅度并量化节点贡献
- 3.4.3 混淆与边界归因：重点分析 S2↔S4 等高混淆场景的代码证据差异
- 3.4.4 有效性讨论：类别不均衡、证据缺失、规则覆盖范围与可迁移性

### 3.5 本章小节
- 总结：分类体系在代码/接口证据驱动下具备可分性与可解释性
- 承接：第四章基于该分类标签展开工具评测与能力边界分析
